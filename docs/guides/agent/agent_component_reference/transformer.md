# 转换器组件

一个使用 LLM 从块中提取见解的组件。

---

**转换器**组件索引块并配置它们在文档引擎中的存储格式。它*通常*位于摄入流水线中的 **索引器**之前,但您也可以按顺序链接多个 **转换器** 组件。

## 场景

当您需要 LLM 从原始块中提取新信息(例如关键字、问题、元数据和摘要)时,**转换器**组件是必不可少的。

## 配置

### 模型

点击 **模型** 的下拉菜单以显示模型配置窗口。

- **模型**: 要使用的聊天模型。
  - 确保您在 **模型提供商** 页面上正确设置了聊天模型。
  - 您可以为不同的组件使用不同的模型,以增加灵活性或提高整体性能。
- **创造性**: **温度**、**Top P**、**存在惩罚** 和 **频率惩罚** 设置的快捷方式,指示模型的自由度。从 **即兴**、**精确** 到 **平衡**,每个预设配置对应 **温度**、**Top P**、**存在惩罚** 和 **频率惩罚** 的独特组合。
  此参数有三个选项:
  - **即兴**: 产生更有创意的响应。
  - **精确**:(默认)产生更保守的响应。
  - **平衡**: **即兴** 和 **精确** 之间的中间地带。
- **温度**: 模型输出的随机性水平。
  默认为 0.1。
  - 较低的值导致更确定性和可预测的输出。
  - 较高的值导致更有创意和多样的输出。
  - 零温度导致相同提示产生相同的输出。
- **Top P**: 核采样。
  - 通过设置阈值 *P* 并将采样限制为累积概率超过 *P* 的标记来减少生成重复或不自然文本的可能性。
  - 默认为 0.3。
- **存在惩罚**: 鼓励模型在响应中包含更多样化的标记范围。
  - 较高的 **存在惩罚** 值导致模型更有可能生成尚未包含在生成文本中的标记。
  - 默认为 0.4。
- **频率惩罚**: 阻止模型在生成的文本中过于频繁地重复相同的单词或短语。
  - 较高的 **频率惩罚** 值导致模型在使用重复标记时更加保守。
  - 默认为 0.7。
- **最大标记数**:
  - 模型的最大上下文大小。

:::tip 注意
- 不必在所有组件中坚持使用相同的模型。如果特定模型在特定任务中表现不佳,请考虑使用不同的模型。
- 如果您不确定 **温度**、**Top P**、**存在惩罚** 和 **频率惩罚** 背后的机制,只需选择 **创造性** 的三个选项之一。
:::

### 结果目标

选择要由 LLM 生成的输出类型:

- 摘要
- 关键字
- 问题
- 元数据

### 系统提示

通常,您使用系统提示来描述 LLM 的任务,指定它应该如何响应,并概述其他杂项要求。我们不打算详细说明此主题,因为它可以像提示工程一样广泛。

:::tip 注意
此处的系统提示会自动更新以匹配您选择的 **结果目标**。
:::

### 用户提示

用户定义的提示。例如,您可以输入 `/` 或点击 **(x)** 来插入摄入流水线中前序组件的变量,作为 LLM 的输入。

### 输出

**转换器**组件输出的全局变量名称,可由摄入流水线中的后续 **转换器** 组件引用。

- 默认值: `chunks`
- 类型: `Array<Object>`
